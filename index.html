Diccionario de elementos HTML5

A continuación se mostraran dde manera exhaustiva todos los elementos que componen el lenguaje de marcado HTML5.

Este diccionario será ordenado de forma alfabetica para un mejor aprendizaje del usuario.

A

<a> Enlace

Crea un enlace a otras páginas de internet, archivos o ubicaciones dentro de la misma página, direcciones de correo, o cualquier otra URL.

<abbr> abreviatura

Representa una abreviación o acrónimo; el atributo opcional title puede ampliar o describir la abreviatura. Si está presente, el atributo title debe contener la descripción completa y nada más.

<address> address

Aporta información de contacto para su article más cercano o ancestro body; en el último caso lo aplica a todo el documento.

<area> area

Sirve para crear las distintas secciones en las que se puede dividir un mapas de imágenes en el cliente.

<article> article

representa una composición auto-contenida en un documento, página, una aplicación o en el sitio, que se destina a distribuir de forma independiente o reutilizable, por ejemplo, en la indicación. Podría ser un mensaje en un foro, un artículo de una revista o un periódico, una entrada de blog, un comentario de un usuario, un widget interactivo o gadget, o cualquier otro elemento independiente del contenido.

<aside> aside

Representa una sección de una página que consiste en contenido que está indirectamente relacionado con el contenido principal del documento. Estas secciones son a menudo representadas como barras laterales o como inserciones y contienen una explicación al margen como una definición de glosario, elementos relacionados indirectamente, como publicidad, la biografía del autor, o en aplicaciones web, la información de perfil o enlaces a blogs relacionados.

<audio> audio

El elemento audio se usa para insertar contenido de audio en un documento HTML o XHTML. El elemento audio se agregó como parte de HTML 5.

<b> bold

Indica que el texto debe ser representado con una variable bold, o negrita, de la tipografía que se esté usando.

<base> bold

Especifica la dirección URL base que se utilizará para todas las direcciones URL relativas contenidas dentro de un documento. Sólo puede haber un elemento "base" en un documento.

<bdi> bi-direccional

Aisla un trozo de texto para que pueda ser formateado con una dirección diferente al texto que hay fuera de él.

<bdo> bi-direccional Overriding

La dirección de escritura del texto está relacionada con el idioma y se declara con el atributo dir. Pero en ocasiones esto no es suficiente, entonces entra en juego el elemento bdo (anulación del algoritmo bidireccional) que al anular el algoritmo permite un control total de la direccionalidad.

<blockqoute> blockquote

Crea citas en bloque, marca las citas a otros autores o documentos.

<body> body

Representa el contenido de un documento HTML. Solo puede haber un elemento "body" en un documento.

<br> line break

Produce un salto de línea en el texto (retorno de carro). Es útil para escribir un poema o una dirección, donde la división de las líneas es significante.

<button> button

Representa un elemento cliqueable de tipo botón que puede ser utilizado en formularios o en cualquier parte de la página que necesite un botón estándar y simple de aplicar. De forma predeterminada, los botones HTML se presentan con un estilo similar en todas las plataformas.

<canvas> canvas

Se puede utilizar para dibujar gráficos a través de secuencias de comandos (por lo general JavaScript ).

<caption> caption

Es el encargado de darle un título descriptivo a las tablas.

<cite> cite

Marca una referencia a una fuente, o el autor de un texto citado.

<code> code

Es el apropiado para marcar el código de un programa.

<col> col

Permite especificar propiedades para una columna o un grupo de ellas.

<colgroup> colgroup

Grupo de columnas. Permite crear grupos de columnas.

<data> data

Vincula un contenido dado con una traducción legible por una máquina.

<datalist> datalist

contiene un conjunto de elementos "option" que representan los valores disponibles para otros controles.

<dd> dd

Provee detalles acerca de o la definición de un término precedente ("dt") en una lista de descripciones ("dl").

<del> deleted

Marca las partes de un texto o documento que han sido suprimidas o sustituidas.

<details> details

Es usado como un widget de revelación a través del cual el usuario puede obtener información adicional .

<dfn> dfn

Sirve para marcar el término que se quiere definir.

<dialog> dialog

Representa una caja de diálogo u otro componente interactivo, como inspector o ventana.

<div> div

Sirve para crear secciones o agrupar contenidos.

<dl> dl

representa una lista descriptiva. El elemento encierra una lista de grupos de términos y de descripciones.

<dt> dt

especifica un término en una descripción o lista de definiciones, y como tal debe utilizarse dentro de un elemento "dl".

<em> em

es el apropiado para marcar con énfasis las partes importantes de un texto.

<embed> embed

Representa un punto de integración para una aplicación externa o de contenido interactivo (en otras palabras, un plug-in).

<fieldset> fieldset

Permite organizar en grupos los campos de un formulario.

<figcaption> figcaption

representa un subtítulo o leyenda asociado al contenido del elemento padre "figure", pudiendo ser colocado como primer o último hijo. Es importante destacar que el elemento "figcaption" es opcional.

<figure> figure

Representa contenido independiente, a menudo con un título. Si bien se relaciona con el flujo principal, su posición es independiente de éste. Por lo general, se trata de una imagen, una ilustración, un diagrama, un fragmento de código, o un esquema al que se hace referencia en el texto principal, pero que se puede mover a otra página o a un apéndice sin que afecte al flujo principal.

<footer> footer

Representa un pie de página para el contenido de sección más cercano o el elemento raíz de sección.

<form> form

Representa una sección de un documento que contiene controles interactivos que permiten a un usuario enviar información a un servidor web.

<head> head

provee información general (metadatos) acerca del documento, incluyendo su título y enlaces a scripts y hojas de estilos.

<header> header

Representa un grupo de ayudas introductorias o de navegación. Puede contener algunos elementos de encabezado, así como también un logo, un formulario de búsqueda, un nombre de autor y otros componentes.

<hr> hr

Representa un cambio de tema entre párrafos (por ejemplo, un cambio de escena en una historia, un cambio de tema en una sección).

<html> html

Representa la raiz de un documento HTML. El resto de elementos descienden de este elemento.

<i> i

Muestra el texto marcado con un estilo en cursiva o italica.

<iframe> iframe

Representa un contexto de navegación anidado, el cual permite incrustrar otra página HTML en la página actual.

<img> img

Representa una imagen en el documento.

<input> input

Se usa para crear controles interactivos para formularios basados en la web con el fin de recibir datos del usuario.

<ins> ins

Marca las partes de un texto que han sido añadidos al documento.

<kbd> kbd

Marca el texto que debe introducir el usuario.

<keygen> keygen

Existe para facilitar la generación de llaves, y el envío de la clave pública como parte de un formulario HTML.

<label> label

Representa una etiqueta para un elemento en una interfaz de usuario. Este puede estar asociado con un control ya sea mediante la utilizacion del atributo for, o ubicando el control dentro del elemento label.

<legend> legend

Crea un título para un grupos los campos ("fieldset") de un formulario.

<li> elemento de lista

Declara cada uno de los elementos de una lista.

<link> link

Especifica la relación entre el documento actual y un recurso externo.

<main> main

Representa el contenido principal del "body" de un documento o aplicación.

<map> map

Descripción de uno o dos párrafos.

<mark> mark

Representa un texto marcado o resaltado como referencia o anotación, debido a su relevancia o importancia en un contexto particular.

<meta> meta

Sirve para aportar información sobre el documento.

<meter> meter

El elemento representa un valor escalar dentro de un rango conocido o un valor fraccionario.

<nav> nav

Representa una sección de una página cuyo propósito es proporcionar enlaces de navegación, ya sea dentro del documento actual o a otros documentos. Ejemplos comunes de secciones de navegación son menús, tablas de contenido e índices.

<noscript> noscript

Aporta contenidos alternativos al elemento script. las aplicaciones de usuario que no soporten scripts deben mostrar en su lugar el contenido de este elemento.

<object> object

Representa un recurso externo, que puede ser tratado como una imagen, un contexto de navegación anidado, o como un recurso que debe ser manejado por un plugin.
